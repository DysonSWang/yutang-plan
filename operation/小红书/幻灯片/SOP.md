# 小红书幻灯片制作 SOP

> **版本**：v4.2
> **日期**：2026-05-11
> **状态**：执行中

---

## 完整工作流

```
选题 → 调研(素材+竞品封面+评论区挖词) → 封面概念 → 文案MD → 评审(封面+正文) → 配图脚本 → 配图制作 → 评审 → 定稿发布 → 数据追踪+迭代
```

## Phase 1 · 选题（基于运营方案）

### 输入
- [小红书起号运营方案.md](../../小红书起号运营方案.md) — 内容系列规划

### 动作
1. 按内容系列比例选择下期主题
2. 确认选题在系列中的位置（第几期）
3. 明确核心信息（一句话总结）

### 示例

| 系列 | 占比 | 下期主题 | 核心信息 |
|------|------|---------|---------|
| 聊天诊断 | 40% | 第1期「在干嘛」 | 状态+好奇比直接回答更能让对方有话可回 |
| 聊天诊断 | 40% | 第2期「聊天总是聊死」 | 3种错误回复都是需求感暴露 |

### 产出
- 确定选题名称
- 写入运营方案的「已执行清单」

## Phase 2 · 调研（素材 + 竞品 + 评论区）

### 输入
- 本地 vault wiki 知识库
- 31位大师方法论
- 小红书平台（竞品笔记）

### 动作

#### 2.1 知识库调研
1. 搜索 vault 中相关素材
   ```bash
   find ~/vault -name "*关键词*" 2>/dev/null
   grep -r "关键词" ~/vault/outputs/ ~/vault/raw/ 2>/dev/null | head -20
   ```
2. 提取核心洞察、案例、金句
3. 匹配对应大师方法论（OS资源）

#### 2.2 竞品封面收集（封面第一法则）
1. 在小红书搜索同主题笔记（3-5篇爆款）
2. 记录爆款封面特征：
   - 标题用什么句式（数字型/冲突型/疑问型）
   - 封面布局（大字+情绪/对比图/真人）
   - 配色和字体
   - 有没有「避坑」「踩雷」「必看」等词

#### 2.3 评论区挖词（评论区是金矿）
1. 找同类型爆款笔记，看评论区
2. 收集用户高频提问/共鸣词
3. 提取这些词用于：
   - 文案中的共鸣点
   - 互动引导的问题来源
   - 标签补充

### 产出
- 素材笔记（3-5条核心引用）
- 对应大师观点（1-2位）
- **竞品封面特征记录（3-5个样本）**
- **评论区高频词清单**

## Phase 2.5 · 封面概念设计（封面第一法则）

> **封面第一法则**：先设计封面和标题，再写正文。正文服务于封面传达的核心信息。

### 输入
- Phase 2 调研结果

### 动作
1. 确定封面标题（从 3 种句式中选）
   - 数字型：「5个技巧」「30天见效」
   - 冲突型：「XX但是XX」「我以为XX结果XX」
   - 疑问型：「为什么XX？」「XX是什么体验」
2. 确定封面视觉方向（参考竞品 + 品牌一致）
3. 写封面概念草案（50字以内钩子 + 核心信息）

### 产出
- 封面标题（定稿前可迭代）
- 封面视觉方向描述
- 封面概念草案

### 输入
- Phase 1 选题

- Phase 2.5 封面概念

### 动作
创建 `[话题名称].md`，包含以下结构：

```markdown

## 话题标签
#[标签1] #[标签2] ...

## 系列归属
- 系列：XXX
- 姊妹篇：[XXX](../XXX/)

## 核心洞察

### 你的独特经历（Dan Koe Personhood）

## 关键词布局
### 标题关键词
### 标签矩阵（5-8个）

**标签规范（起号阶段）**：
- ✅ 可用：`#聊天技巧 #情感共鸣 #两性关系 #沟通艺术 #废物测试 #送命题 #王者思维 #高情商回复`

**标准标签矩阵公式**（5-8个）：
```

```

**示例**（聊天诊断系列）：
```

## Phase 5 · 配图脚本（HTML幻灯片）

### 输入
- 定稿文案MD
- 封面概念（Phase 2.5）
- 设计系统规范（见下方）

### 动作
1. 按封面概念先做 Slide 1（封面）
2. 确认封面视觉方向后，再做 Slide 2-7
3. 引用统一设计系统（色值、字体、尺寸）

### 设计系统规范

**色彩**：
| 用途 | 色值 |
|------|------|
| 背景-暖色 | `#FFF8F3` |
| 背景-深色 | `#0d0d1a → #1a1a2e` |
| 主色-珊瑚红 | `#E85A4F` |
| 强调-墨绿 | `#3D8C40` |
| 点缀-古铜金 | `#C9A227` |
| 正文-墨色 | `#2D2D2D` |

**字体**：
| 用途 | 字体 |
|------|------|
| 标题 | Noto Serif SC（思源宋体） |
| 正文 | Noto Sans SC（无衬线） |

**尺寸**：
| 元素 | 尺寸 |
|------|------|
| 画布 | 540×720px（3:4竖版） |

**Footer规范**：
| 属性 | 规则 | 错误值 |
|------|------|--------|
| `bottom` | ≥ 30px | 14px / 18px（会被截图截掉） |
| `border-radius` | ≤ 10px | 20px（超出overflow:hidden被裁） |
| `color` | `var(--ink)` (#2D2D2D) | `#6B6B6B`（太浅看不清） |
| `font-size` | ≥ 14px | 12px |
| `font-weight` | ≥ 600 | 400 |
| `background` | `rgba(255,255,255,0.85)` | 无白底（对比度不够） |

**封面标题规范（按渲染像素宽度计算）**：

画布宽度 540px，标题区左右各留 30px padding，可用宽度约 480px。

| 字数 | 字号 | `white-space` | 说明 |
|------|------|---------------|------|
| ≤ 10字 | ≥ 58px | `nowrap` | 短标题大字冲击 |
| 11-13字 | 34-38px | 去掉 | 允许换行，字号随字数递减 |
| 14-16字 | 28-32px | 去掉 | 长标题需缩小字号，允许2行 |
| > 16字 | ⚠️ 重新简化 | — | 超过16字无法在封面完整显示，必须简化标题 |

**关键原理**：中文字宽约等于字号（含间距约字号+10px）。
- 58px × 11字 ≈ 638px > 480px → 溢出 ❌
- 34px × 13字 ≈ 442px < 480px → 安全 ✅
- 38px × 14字 ≈ 532px > 480px → 溢出 ❌

**实操**：写封面标题时先数字数，超过 10 字必须缩小字号 + 去掉 nowrap。超过 16 字必须改短标题。

**截图预览规则**：
- 每次修改后必须截图预览，确认：标题完整显示、footer不被截
- 截图后必须在手机端实际查看，不能只看桌面预览
- **先截图封面确认无误，再做其余6张**

### 2026-05-10 内容排版被截图截断踩坑

**问题**：slide-06 内容过多，think 区域和 bonus 区域溢出导致底部被截图截掉。

**修复**：整体上移排版，压缩顶部和中间间距。

| `.block` margin | 6px | 8px/10px | 区块间距压缩防溢出 |
| `.block` padding | 14px | 16px/18px | 内边距压缩 |
| `.block-title` font-size | 17px | 18px/20px | 标题字号微缩 |
| `.block-example` font-size | 13px | 14px | 示例文字微缩 |
| `.bonus` margin | 6px | 10px/14px | bonus区间距压缩 |
| `.bonus` padding | 10px 16px | 12px 18px | bonus区内边距压缩 |
| `.bonus-text` font-size | 13px | 14px | 文字微缩 |
| `.bonus-answer` font-size | 13px | 14px | 文字微缩 |
| `.bonus-answer` margin-top | 3px | 4px | 间距微缩 |
| `.think` margin-bottom | 0 | 100px | 底部不留大空白 |
| `.think` margin-top | 6px | 10px/14px | 间距压缩 |
| `.think` padding | 12px 16px | 14px 18px | 内边距压缩 |
| `.subtitle` margin-top | 2px | 4px | 副标题间距压缩 |
| `.divider` margin-top | 4px | 8px | 分割线间距压缩 |
| footer 空 div | 删除 | 保留 | 空 footer 占用空间无意义 |

**坑页（slide-03/04/05）间距规则不同**：
- `.title` padding-top: 12px（有2个section+1个think，内容少不用压缩）
- `.section` margin: 10px，padding: 16px 18px
- `.think` margin: 14px 20px，padding: 16px 18px

## Phase 9 · 数据追踪与迭代

### 数据追踪

| 指标 | 追踪时间 | 判定标准 |
|------|---------|---------|
| 曝光量 | 发布后 24h / 72h / 7d | 200-500 → 正常；<200 → 限流可能 |
| 互动率 | 发布后 24h / 72h | >5% → 进下一池；>8% → 爆款潜质 |
| 收藏率 | 发布后 7d | 高 = 搜索流量长尾 |
| 评论数 | 发布后 24h | >10 → 算法会重新扩大推荐 |

### 判定与动作

| 数据表现 | 判定 | 动作 |
|---------|------|------|
| 互动率>8% | 爆款潜质 | **爆款复制**：趁热度发同系列2-3篇 |
| 互动率5-8% | 正常 | 回复所有评论，延长推荐周期 |
| 互动率<5% | 偏弱 | 分析：封面？选题？还是关键词？ |
| 曝光<200（连续7天） | 限流可能 | 检查：标签是否违规？内容是否触碰红线？ |
| 连续30天曝光<500 | 死号判断线 | 重新审视定位或考虑放弃 |

### 迭代决策

**爆款复制放大（启发式第5条）**：
- 数据好的笔记，趁热度发同系列3篇
- 封面风格保持一致
- 标题句式复用

**转型小步测试（启发式第6条）**：
- 想换方向 → 先发5-10篇新方向内容试水
- 给3个月观察期再决定

### 产出
- 数据记录
- 迭代决策（复制/调整/放弃/转型）

## 批量生产机制

### 适用场景
运营方案要求快速起号（Day 1-6 发30篇）时，单篇串行来不及，需要并行批量生产。

### 方法

#### 方法A：同系列批量做封面概念
1. 一次定 5 个同系列选题
2. 统一调研，一次收集竞品封面 + 评论区词
3. 批量写 5 个封面概念
4. 评审通过后，批量写文案MD

#### 方法B：模板复用
1. 确定一套设计模板（7张幻灯片结构）
2. 只换内容，不换结构
3. 用 SOP 里的「7张模板结构」快速填充

### 产出
- 批量选题清单（5-10个）
- 批量封面概念（5-10个）
- 批量文案MD（5-10个）

## 设计踩坑记录

### 2026-05-09 「在干嘛」评审发现

| 问题 | 影响 | 修复 |
|------|------|------|
| 封面下半部留白过多 | 视觉空洞，信息量不足 | 加对话预览「她：在干嘛？→你：在上班→她：哦→话题结束」填充 |
| 封面CTA文字偏淡偏小 | 手机上看不清 | 字号12→14px，颜色从ink-light→coral加粗 |
| 封面三个装饰点无意义 | 浪费空间 | 移除，替换为对话预览 |
| Slide 2 段位标签字号偏小 | 手机上看不清 | 11px → 13px |
| 底部footer文字普遍偏小 | 可读性差 | 12px → 13px |

**通用规则**（后续所有HTML模板必须遵守）：
- 标签/编号字号 ≥ 13px
- Footer 文字：浅色页 `color: var(--ink)` + `font-weight: 600` + ≥ 14px；深色页 `rgba(255,255,255,0.75)` 以上
- CTA文字不用淡灰色，用品牌色（珊瑚红/墨绿）加粗
- 封面不能有纯装饰性无信息元素
- 封面主标题：文字短（≤12字）用 ≥ 58px + `white-space: nowrap`；文字长（>12字）用 34-40px + 去掉 `white-space: nowrap` 允许换行
- 所有文字在手机端必须清晰可读（对比度 ≥ 4.5:1）

### 2026-05-10 起号阶段禁止内容规范

**起号阶段（粉丝少、影响力弱）禁止出现的诱导互动内容**：

| 禁止内容 | 原因 | 正确做法 |
|---------|------|---------|
| `下期预告：xxx` | 没有下期系列时不要写 | 用 `收藏起来，聊天直接用` |
| `点赞 + 评论"xxx" 领完整版` | 起号阶段没有影响力，无法兑现 | 删除整行，或只保留 `👍 点赞 + 评论"xxx"` |
| 任何需要用户操作才能获取的内容暗示 | 容易让用户觉得是套路 | 专注内容价值，让用户自然互动 |

**执行命令**：
```bash
# 删除"下期预告"整行

# 删除"领完整版"整行
```

### 2026-05-09 手机端实际预览发现问题

**问题来源**：通过DingTalk截图在手机端实际预览「聊天总是聊死」第二集，发现以下问题：

| 问题 | 影响 | 修复 |
|------|------|------|
| 封面标题字太小（50px） | 信息流中被其他笔记淹没 | 封面主标题 ≥ 58px，加 `white-space: nowrap` 防换行 |
| 底部footer文字用浅灰 `#6B6B6B` | 手机上几乎看不见 | footer 必须用深色 `#2D2D2D` + 加粗 + ≥ 14px |
| 深色页说明文字 `rgba(255,255,255,0.5)` | 在深色背景上太淡 | 深色页文字 ≥ `rgba(255,255,255,0.75)` |

**手机端可读性硬性标准**（后续所有HTML模板必须遵守）：

1. **底部文字**：颜色必须用 `var(--ink)` (#2D2D2D)，字号 ≥ 14px，`font-weight: 600`
2. **封面标题**：≤10字用 ≥58px+nowrap；11-13字用 34-38px+允许换行；14-16字用 28-32px+允许换行；>16字必须简化标题
3. **深色页文字**：任何在深色背景上的文字，alpha ≥ 0.75（禁止 0.5 以下）
4. **浅色页正文**：所有辅助说明文字不用 `var(--ink-light)`，用 `var(--ink)` 或 `rgba(0,0,0,0.6)`
5. **最小字号**：正文 ≥ 14px，标签 ≥ 13px

### 2026-05-11 公式页示例匹配 · 黑话翻译 · 一致性检查

**问题来源**：第五篇「约不出来怎么办」slide-06 公式页制作过程中发现三类问题。

---

**问题1：公式/方法的示例和标题不对应**

| 错误案例 | 原因 | 正确做法 |
|---------|------|---------|
| 方式二「线上建立吸引」，示例配了「那家店很不错，改天带你去」| 示例是约见面，不是线上建立吸引 | 示例要体现"不依赖见面"的本质动作 |
| 方式三「状态回调」，示例用了模糊邀约的话术 | 状态回调 = 换话题等窗口，不是推迟邀约 | 示例要体现"不纠缠，聊别的"的动作 |

**规则**：每个公式/方法的标题 → 必须配一句**具体话术示例**，不能只做概念描述。示例要体现这个方法"具体怎么说/怎么做"。

---

**问题2：圈内黑话需要翻译**

| ❌ 圈内黑话 | ✅ 公共渠道说法 | 原因 |
|------------|---------------|------|
| 外卖方法 | 线上建立吸引 | 小红书用户不懂"外卖"是什么意思 |
| IoD信号 | 拒绝/没兴趣的信号 | 缩写外行看不懂 |
| 框架 | 主动权/框架（保留，普通人能猜到意思）| 可保留但尽量用大白话 |
| 需求感 | 低价值/暴露需求 | 圈内人能理解，公众不太懂，可保留 |

**规则**：发到公共渠道（小红书/抖音/公众号）之前，**所有圈内高频词/黑话必须翻译成人话**。

**适用范围**：HTML幻灯片 + 文案MD + 发布文案.md，全部都要检查。

---

**问题3：slide-06 和 slide-07 内容不一致**

| 位置 | 互相不一致的表现 |
|------|----------------|
| 方式二示例 | slide-06 改完后，slide-07 还是旧版 |
| 方式三示例 | 同上 |

**规则**：slide-06 写完必须同步到 slide-07 总结页。全部写完后做"一致性横向扫描"：对比所有引用同一个内容的地方是否一致。

---

**问题4：示例要有具体话术，不能只有描述**

| ❌ 只有描述 | ✅ 有具体话术 |
|-----------|-------------|
| 「线上完成价值展示」 | 「发高质量朋友圈、聊天展示魅力 → 她在线上就对你上头」 |
| 「换时间换话题」 | 「她拒绝后不纠缠，聊别的话题 → 等她主动提」 |

**规则**：公式页示例 = **一句话展示具体怎么说**，描述动作不等于示例。
