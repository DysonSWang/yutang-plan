# 小红书幻灯片制作 SOP

> **版本**：v3.0
> **日期**：2026-05-09
> **状态**：执行中

---

## 完整工作流

```
选题 → 调研(素材+竞品封面+评论区挖词) → 封面概念 → 文案MD → 评审(封面+正文) → 配图脚本 → 配图制作 → 评审 → 定稿发布 → 数据追踪+迭代
```

## Phase 1 · 选题（基于运营方案）

### 输入
- [小红书起号运营方案.md](../../小红书起号运营方案.md) — 内容系列规划

### 动作
1. 按内容系列比例选择下期主题
2. 确认选题在系列中的位置（第几期）
3. 明确核心信息（一句话总结）

### 示例

| 系列 | 占比 | 下期主题 | 核心信息 |
|------|------|---------|---------|
| 聊天诊断 | 40% | 第1期「在干嘛」 | 状态+好奇比直接回答更能让对方有话可回 |
| 聊天诊断 | 40% | 第2期「聊天总是聊死」 | 3种错误回复都是需求感暴露 |

### 产出
- 确定选题名称
- 写入运营方案的「已执行清单」

## Phase 2 · 调研（素材 + 竞品 + 评论区）

### 输入
- 本地 vault wiki 知识库
- 31位大师方法论
- 小红书平台（竞品笔记）

### 动作

#### 2.1 知识库调研
1. 搜索 vault 中相关素材
   ```bash
   find ~/vault -name "*关键词*" 2>/dev/null
   grep -r "关键词" ~/vault/outputs/ ~/vault/raw/ 2>/dev/null | head -20
   ```
2. 提取核心洞察、案例、金句
3. 匹配对应大师方法论（OS资源）

#### 2.2 竞品封面收集（封面第一法则）
1. 在小红书搜索同主题笔记（3-5篇爆款）
2. 记录爆款封面特征：
   - 标题用什么句式（数字型/冲突型/疑问型）
   - 封面布局（大字+情绪/对比图/真人）
   - 配色和字体
   - 有没有「避坑」「踩雷」「必看」等词

#### 2.3 评论区挖词（评论区是金矿）
1. 找同类型爆款笔记，看评论区
2. 收集用户高频提问/共鸣词
3. 提取这些词用于：
   - 文案中的共鸣点
   - 互动引导的问题来源
   - 标签补充

### 产出
- 素材笔记（3-5条核心引用）
- 对应大师观点（1-2位）
- **竞品封面特征记录（3-5个样本）**
- **评论区高频词清单**

## Phase 2.5 · 封面概念设计（封面第一法则）

> **封面第一法则**：先设计封面和标题，再写正文。正文服务于封面传达的核心信息。

### 输入
- Phase 2 调研结果

### 动作
1. 确定封面标题（从 3 种句式中选）
   - 数字型：「5个技巧」「30天见效」
   - 冲突型：「XX但是XX」「我以为XX结果XX」
   - 疑问型：「为什么XX？」「XX是什么体验」
2. 确定封面视觉方向（参考竞品 + 品牌一致）
3. 写封面概念草案（50字以内钩子 + 核心信息）

### 产出
- 封面标题（定稿前可迭代）
- 封面视觉方向描述
- 封面概念草案

## Phase 4 · 多轮评审文案

### 评审维度

| 维度 | 检查点 | 对应模型 |
|------|--------|---------|
| **封面第一** | 封面标题有冲突感吗？在信息流能抓住眼球吗 | 爆款四要素-视觉冲击 |
| **情绪强度** | 看开头会不会「心里一震」 | 爆款四要素-情绪强度 |
| **实用价值** | 读者能得到一个行动指令吗 | 爆款四要素-实用价值 |
| **真实感** | 有没有个人经历，不像广告 | 模型1真实感原力 |
| **Dan Koe框架** | Problem→Process→Personhood 是否完整 | Dan Koe方法论 |
| **互动引导** | 用提问还是指令？是否自然 | 模型4互动率 |
| **搜索SEO** | 关键词布局是否合理 | 模型5搜索心智 |

### 动作
1. 先评封面（封面第一法则）
2. 用 xiaohongshu-framework skill 评审
3. 用 dan-koe-perspective skill 评审
4. 用 love-panel-perspective（情感大师团）评审
5. 根据反馈修改文案MD

### 产出
- 文案MD定稿（标记「定稿」状态）

## Phase 5 · 配图脚本（HTML幻灯片）

### 输入
- 定稿文案MD
- 封面概念（Phase 2.5）
- 设计系统规范（见下方）

### 动作
1. 按封面概念先做 Slide 1（封面）
2. 确认封面视觉方向后，再做 Slide 2-7
3. 引用统一设计系统（色值、字体、尺寸）

### 设计系统规范

**色彩**：
| 用途 | 色值 |
|------|------|
| 背景-暖色 | `#FFF8F3` |
| 背景-深色 | `#0d0d1a → #1a1a2e` |
| 主色-珊瑚红 | `#E85A4F` |
| 强调-墨绿 | `#3D8C40` |
| 点缀-古铜金 | `#C9A227` |
| 正文-墨色 | `#2D2D2D` |

**字体**：
| 用途 | 字体 |
|------|------|
| 标题 | Noto Serif SC（思源宋体） |
| 正文 | Noto Sans SC（无衬线） |

**尺寸**：
| 元素 | 尺寸 |
|------|------|
| 画布 | 540×720px（3:4竖版） |

### 7张模板结构

| Slide | 作用 | 背景 | 关键元素 |
|-------|------|------|---------|
| 1 · 封面 | 吸引点击 | 暖色 | 珊瑚红标签+衬线标题+金色金句 |
| 2 · 场景钩子 | 引起共鸣 | 深色 | 毛玻璃卡片+分级展示 |
| 3 · 对比 | 展示差异 | 暖色 | 青铜vs王者对比 |
| 4 · 公式 | 给方案 | 暖色 | 墨绿标签+公式框 |
| 5 · 思维 | 讲为什么 | 暖色 | 金色标签+编号列表 |
| 6 · 进阶 | 提升技巧 | 深色 | 绿色标题栏+技巧卡片 |
| 7 · CTA | 互动引导 | 深色 | 公式总结+提问 |

### 产出
- `[话题名称]/幻灯片/slide-01-cover.html`
- `[话题名称]/幻灯片/slide-02-scenarios.html`
- ...（共7张）

## Phase 9 · 数据追踪与迭代

### 数据追踪

| 指标 | 追踪时间 | 判定标准 |
|------|---------|---------|
| 曝光量 | 发布后 24h / 72h / 7d | 200-500 → 正常；<200 → 限流可能 |
| 互动率 | 发布后 24h / 72h | >5% → 进下一池；>8% → 爆款潜质 |
| 收藏率 | 发布后 7d | 高 = 搜索流量长尾 |
| 评论数 | 发布后 24h | >10 → 算法会重新扩大推荐 |

### 判定与动作

| 数据表现 | 判定 | 动作 |
|---------|------|------|
| 互动率>8% | 爆款潜质 | **爆款复制**：趁热度发同系列2-3篇 |
| 互动率5-8% | 正常 | 回复所有评论，延长推荐周期 |
| 互动率<5% | 偏弱 | 分析：封面？选题？还是关键词？ |
| 曝光<200（连续7天） | 限流可能 | 检查：标签是否违规？内容是否触碰红线？ |
| 连续30天曝光<500 | 死号判断线 | 重新审视定位或考虑放弃 |

### 迭代决策

**爆款复制放大（启发式第5条）**：
- 数据好的笔记，趁热度发同系列3篇
- 封面风格保持一致
- 标题句式复用

**转型小步测试（启发式第6条）**：
- 想换方向 → 先发5-10篇新方向内容试水
- 给3个月观察期再决定

### 产出
- 数据记录
- 迭代决策（复制/调整/放弃/转型）

## 批量生产机制

### 适用场景
运营方案要求快速起号（Day 1-6 发30篇）时，单篇串行来不及，需要并行批量生产。

### 方法

#### 方法A：同系列批量做封面概念
1. 一次定 5 个同系列选题
2. 统一调研，一次收集竞品封面 + 评论区词
3. 批量写 5 个封面概念
4. 评审通过后，批量写文案MD

#### 方法B：模板复用
1. 确定一套设计模板（7张幻灯片结构）
2. 只换内容，不换结构
3. 用 SOP 里的「7张模板结构」快速填充

### 产出
- 批量选题清单（5-10个）
- 批量封面概念（5-10个）
- 批量文案MD（5-10个）

## 目录结构

```
幻灯片/
├── SOP.md                    ← 本文件
└── [话题名称]/
    ├── 配图/                  ← PNG截图
    ├── 幻灯片/                ← HTML源文件
    ├── 文案/                  ← MD文案
```

**命名规范**：
- HTML：`slide-01-cover.html` ~ `slide-07-cta.html`
- PNG：`slide-01-cover.png` ~ `slide-07-cta.png`
- 文案：`[话题名称].md`

## 设计踩坑记录

### 2026-05-09 「在干嘛」评审发现

| 问题 | 影响 | 修复 |
|------|------|------|
| 封面下半部留白过多 | 视觉空洞，信息量不足 | 加对话预览「她：在干嘛？→你：在上班→她：哦→话题结束」填充 |
| 封面CTA文字偏淡偏小 | 手机上看不清 | 字号12→14px，颜色从ink-light→coral加粗 |
| 封面三个装饰点无意义 | 浪费空间 | 移除，替换为对话预览 |
| Slide 2 段位标签字号偏小 | 手机上看不清 | 11px → 13px |
| 底部footer文字普遍偏小 | 可读性差 | 12px → 13px |

**通用规则**（后续所有HTML模板必须遵守）：
- 标签/编号字号 ≥ 13px
- Footer字号 ≥ 13px
- CTA文字不用淡灰色，用品牌色（珊瑚红/墨绿）加粗
- 封面不能有纯装饰性无信息元素

### 硬性规则：统一以「我」的口吻

**所有幻灯片必须用「我」的第一人称口吻，严禁出现任何导师/他人名字。**

| ❌ 禁止 | ✅ 正确写法 |
|--------|-----------|
| Leon说：不追发是天条 | 我的底线：不追发是天条 |
| Ryan说：情绪流动是关键 | 我的体会：情绪流动才是关键 |
| 大迪的底层逻辑是... | 我总结的底层逻辑是... |
| 下期预告：如何自然邀约 | 下期预告：[计划中的下一期标题] |

**执行要点**：
- 所有金句/原则/底线 → 改为「我的××：」
- 所有下期预告 → 必须对照运营方案中的系列计划，不能自创
- HTML交付前必须 `grep -rn "Leon\|Ryan\|大迪\|林老头"` 全文扫描
