## 完整工作流

```
选题 → 调研(素材+竞品封面+评论区挖词) → 封面概念 → 文案MD → 评审(封面+正文) → 配图脚本 → 配图制作 → 评审 → 定稿发布 → 数据追踪+迭代
```

## Phase 1 · 选题（基于运营方案）

### 输入
- [小红书起号运营方案.md](../../小红书起号运营方案.md) — 内容系列规划

### 动作
1. 按内容系列比例选择下期主题
2. 确认选题在系列中的位置（第几期）
3. 明确核心信息（一句话总结）

### 示例

| 系列 | 占比 | 下期主题 | 核心信息 |
|------|------|---------|---------|
| 聊天诊断 | 40% | 第1期「在干嘛」 | 状态+好奇比直接回答更能让对方有话可回 |
| 聊天诊断 | 40% | 第2期「聊天总是聊死」 | 3种错误回复都是需求感暴露 |

### 产出
- 确定选题名称
- 写入运营方案的「已执行清单」

## Phase 2 · 调研（素材 + 竞品 + 评论区）

### 输入
- 本地 vault wiki 知识库
- 31位大师方法论
- 小红书平台（竞品笔记）

### 动作

#### 2.1 知识库调研
1. 搜索 vault 中相关素材
   ```bash
   find ~/vault -name "*关键词*" 2>/dev/null
   grep -r "关键词" ~/vault/outputs/ ~/vault/raw/ 2>/dev/null | head -20
   ```
2. 提取核心洞察、案例、金句
3. 匹配对应大师方法论（OS资源）

#### 2.2 竞品封面收集（封面第一法则）
1. 在小红书搜索同主题笔记（3-5篇爆款）
2. 记录爆款封面特征：
   - 标题用什么句式（数字型/冲突型/疑问型）
   - 封面布局（大字+情绪/对比图/真人）
   - 配色和字体
   - 有没有「避坑」「踩雷」「必看」等词

#### 2.3 评论区挖词（评论区是金矿）
1. 找同类型爆款笔记，看评论区
2. 收集用户高频提问/共鸣词
3. 提取这些词用于：
   - 文案中的共鸣点
   - 互动引导的问题来源
   - 标签补充

### 产出
- 素材笔记（3-5条核心引用）
- 对应大师观点（1-2位）
- **竞品封面特征记录（3-5个样本）**
- **评论区高频词清单**

## Phase 2.5 · 封面概念设计（封面第一法则）

> **封面第一法则**：先设计封面和标题，再写正文。正文服务于封面传达的核心信息。

### 输入
- Phase 2 调研结果

### 动作
1. 确定封面标题（从 3 种句式中选）
   - 数字型：「5个技巧」「30天见效」
   - 冲突型：「XX但是XX」「我以为XX结果XX」
   - 疑问型：「为什么XX？」「XX是什么体验」
2. 确定封面视觉方向（参考竞品 + 品牌一致）
3. 写封面概念草案（50字以内钩子 + 核心信息）

### 产出
- 封面标题（定稿前可迭代）
- 封面视觉方向描述
- 封面概念草案

## Phase 5 · 配图脚本（HTML幻灯片）

### 输入
- 定稿文案MD
- 封面概念（Phase 2.5）
- 设计系统规范（见下方）

### 动作
1. 按封面概念先做 Slide 1（封面）
2. 确认封面视觉方向后，再做 Slide 2-7
3. 引用统一设计系统（色值、字体、尺寸）

### 设计系统规范

**色彩**（2026-05-11 更新，必须同步改 RGBA 硬编码）：
| 用途 | 色值 | RGBA 硬编码（需同步） |
|------|------|----------------------|
| 背景-暖色 | `#FFF8F3` | — |
| 背景-深色 | `#0d0d1a → #1a1a2e` | `rgba(13,13,26,1) / rgba(26,26,46,1)` |
| 主色-珊瑚红 | `#C0392B`（暖白背景） / `#E74C3C`（深色背景） | 深色背景用 `0.15-0.2` 透明度 |
| 强调-墨绿 | `#3D8C40`（暖白背景） / `#27AE60`（深色背景） | 深色背景用 `0.1-0.15` 透明度 |
| 点缀-古铜金 | `#A08020`（暖白背景） / `#F39C12`（深色背景） | 深色背景用 `0.06-0.1` 透明度 |
| 正文-墨色 | `#2D2D2D` | — |
| 次要文字 | `#4A4A4A` | — |

> **背景适配规则**：深色背景（slide-02/07）用亮色（`#E74C3C`/`#27AE60`/`#F39C12`），暖白背景用暗色（`#C0392B`/`#3D8C40`/`#A08020`）。深色背景对比度要求更高，暗色在深色上对比度不足。
> **RGBA 硬编码随动规则**：CSS 变量 `--coral` / `--gold` / `--green` 改值时，所有直接写在 HTML/CSS 中的 `rgba(192,57,43,...)` / `rgba(160,128,32,...)` / `rgba(61,140,64,...)` 必须同步更新。不能引用变量的 RGBA 值改一处全改。**正确做法是：只在 `:root` 里定义主色变量，RGBA 值通过 `color-mix(in srgb, var(--coral) 8%, transparent)` 或手动同步。**

**字体**：
| 用途 | 字体 |
|------|------|
| 标题 | Noto Serif SC（思源宋体） |
| 正文 | Noto Sans SC（无衬线） |

**尺寸**：
| 元素 | 尺寸 |
|------|------|
| 画布 | 540×720px（3:4竖版） |

**Footer规范**：
| 属性 | 规则 | 错误值 |
|------|------|--------|
| `bottom` | ≥ 30px | 14px / 18px（会被截图截掉） |
| `border-radius` | ≤ 10px | 20px（超出overflow:hidden被裁） |
| `color` | `var(--ink)` (#2D2D2D) | `#6B6B6B`（太浅看不清） |
| `font-size` | ≥ 14px | 12px |
| `font-weight` | ≥ 600 | 400 |
| `background` | `rgba(255,255,255,0.85)` | 无白底（对比度不够） |

**封面标题规范（按渲染像素宽度计算）**：

画布宽度 540px，标题区左右各留 30px padding，可用宽度约 480px。

| 字数 | 字号 | `white-space` | 说明 |
|------|------|---------------|------|
| ≤ 10字 | ≥ 58px | `nowrap` | 短标题大字冲击 |
| 11-13字 | 34-38px | 去掉 | 允许换行，字号随字数递减 |
| 14-16字 | 28-32px | 去掉 | 长标题需缩小字号，允许2行 |
| > 16字 | ⚠️ 重新简化 | — | 超过16字无法在封面完整显示，必须简化标题 |

**关键原理**：中文字宽约等于字号（含间距约字号+10px）。
- 58px × 11字 ≈ 638px > 480px → 溢出 ❌
- 34px × 13字 ≈ 442px < 480px → 安全 ✅
- 38px × 14字 ≈ 532px > 480px → 溢出 ❌

**实操**：写封面标题时先数字数，超过 10 字必须缩小字号 + 去掉 nowrap。超过 16 字必须改短标题。

**截图预览规则**：
- 每次修改后必须截图预览，确认：标题完整显示、footer不被截
- 截图后必须在手机端实际查看，不能只看桌面预览
- **先截图封面确认无误，再做其余6张**

### 7张模板结构

| Slide | 作用 | 背景 | 关键元素 | 视觉目标 |
|-------|------|------|---------|---------|
| 1 · 封面 | 吸引点击 | 暖色→珊瑚红Banner | 大字衬线标题+弧形底部裁切 | 标签按内容本质选择：危险动作→"雷区"，危险提问→"送命题" |
| 2 · 场景钩子 | 引起共鸣 | 深色玻璃 | 毛玻璃卡片+分级展示 | 让用户"是我"的共鸣感 |
| 3 · 对比 | 展示差异 | 暖色 | 青铜vs王者对比 | 建立"我踩过这个坑"的认知 |
| 4 · 公式 | 给方案 | 暖色 | 墨绿标签+公式框 | 给用户收藏理由（有用） |
| 5 · 思维 | 讲为什么 | 暖色 | 金色标签+编号列表 | 建立信任感（不只是技巧） |
| 6 · 进阶 | 提升技巧 | 暖色 | 绿色标签+公式框 | 给进阶用户更多价值 |
| 7 · CTA | 互动引导 | 深色 | **核心认知语+讨论区（无方法重复）** | **引导评论，降低参与门槛** |

> **⚠️ 6、7 分工铁律（防止内容重复）**：
> - **Slide-06**：给方案 → 详细展开3种方法/公式，每种配具体话术示例
> - **Slide-07**：给认知 → 只放核心金句 + 讨论区，**不重复6的方法内容**
> - 错误模式：6和7都写"3种方法"→ 用户看一遍就够了，第二页多余
> - 正确模式：6给完整方案 → 7给认知升华（为什么对）+ 行动召唤（评论）

> **封面视觉冲击力规范（2026-05-11 更新）**：封面是信息流中唯一能停滑手的元素。设计必须遵循：
> - **珊瑚红 Banner 上撑**：标题文字嵌入珊瑚红色块中，白底留白衬托，最大化色块面积
> - **大字标题 ≥ 50px**：用衬线体（`Noto Serif SC`），白色或深底色确保对比度
> - **弧形裁切分界**：Banner 底部用 `clip-path: ellipse(55% 100% at 50% 100%)` 制造柔和过渡，替代直角分割线
> - **禁止纯装饰**：去掉 deco 圆/散点等无效装饰，让文字本身成为视觉锚点
> - **顶部标签选择规则**：
>   - **"雷区"**：内容是**动作/行为**——做了就死（问原因、强行说服、直接放弃）
>   - **"送命题"**：内容是**提问/话题**——答了就暴露（你喜欢我什么、你谈过几个）
>   - **"致命"**：更重的警告，用于高危险动作
>   - **"禁区"**：效果同"雷区"，风格更硬
>   - **禁止用"操作类问题"**：这个词太中性，不传达危险感
> - **CTA 用珊瑚红实心按钮**：白色背景+圆角文字的 CTA 在信息流里不够显眼，应用珊瑚红填充按钮

> **评论区引导规范（2026-05-11 更新）**：
> - 结尾页（slide-07）必须有讨论区，设计为"带示范的提问"：
>   - 写一句"我先说：..."作为用户评论的示范
>   - 降低用户组织语言的门槛 → 提高评论数 → 提升算法推荐权重
> - 讨论区背景用 `rgba(255,255,255,0.04)` 浅色块，与卡片视觉区分
> - CTA 引导词用 `说出来，帮你分析 ↓` 而不是泛泛的"评论区告诉我"

**字体规范（铁律，所有页面必须统一）**：

| 元素 | 字号 | 粗细 | 符号 |
|------|------|------|------|
| 标签（如"青铜"/"王者"前缀）| 14px | 700 | 大写英文或符号 |
| 副标题 | 15px | 500 | — |
| 小标题 | 17px | 700 | — |
| 正文 | 15px | 400 | — |
| footer文字 | 14px | 700 | — |

**最小字号限制（防止模糊，提高可读性）**：

| 类型 | 最小字号 | 说明 |
|------|---------|------|
| 标题（封面/章节） | 28px | 28px以下在手机端看不清 |
| 正文（对话/描述） | 15px | 15px以下模糊 |
| 标签（X/V/方式标签） | 14px | 14px以下看不清 |
| 说明文字（副标题/注释） | 13px | 13px以下模糊 |

**符号体系（铁律，禁止混用）**：
| 含义 | 符号 | 示例 |
|------|------|------|
| 错误/青铜 | `X` | `X 青铜` |
| 正确/王者 | `V` | `V 王者` |
| 结果标记 | `!` | `! 结果` |
| 结果区分 | `◆` | `◆ 结果` |

> ⚠️ **禁止**：✗ ✓ ❌ ✅ ❓ ⚠️ 等异体符号混入，必须统一。

**Footer CSS 语法规范**：
```css
.footer span {
  cursor: pointer;
  transition: transform 0.15s ease;
  font-size: 14px;
  font-weight: 700;
  padding: 4px 16px;
  border-radius: 10px;
  background: rgba(255,255,255,0.88);
  color: var(--ink);
}
.footer span:active { transform: scale(0.96); }
```
> ⚠️ **禁止**：把 `font-size` 等属性写在 `:active` 同一行后面，这会导致语法错误使整条规则失效。

## Phase 5.5 · HTML Design Review（design-taste-frontend 框架）

> **触发时机**：每次 HTML 写完、截图之前，必须执行此评审。

### 设计基准参数
```
DESIGN_VARIANCE: 8  (1=对称 10=艺术感)
MOTION_INTENSITY: 1  (1=静态 10=电影感)
VISUAL_DENSITY:  4  (1=留白 10=密集)
```
本项目是小红书竖版幻灯片，属于「静态 + 中等密度 + 偶有艺术感」区间。

---

### 评审清单（按优先级）

#### P0 · 可访问性（CRITICAL）

| 检查项 | 工具/方法 | 达标标准 |
|--------|----------|---------|
| 颜色对比度 | 目测 + [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/) | 正文在背景上 ≥ 4.5:1；大型文本 ≥ 3:1 |
| **珊瑚红 `#E85A4F` → `#C0392B`** | 搜索 `#E85A4F` | 珊瑚色在 `#FFF8F3` 上对比度仅 3.2:1，必须改为 `#C0392B`（4.8:1） |
| **金色 `#C9A227` → `#A08020`** | 搜索 `#C9A227` | 金色在 `#FFF8F3` 上对比度仅 3.1:1，必须改为 `#A08020`（5.2:1） |
| 深色背景文字 | `rgba(255,255,255,0.5)` | 深色背景上文字 alpha 必须 ≥ 0.75 |

**快速自检命令**：
```bash
# 检查旧珊瑚色
grep -rn "#E85A4F\|rgba(232,90,79" "$dir"/slide-*.html

# 检查旧金色
grep -rn "#C9A227\|rgba(201,162,39" "$dir"/slide-*.html

# 检查 emoji
grep -rn "👑\|💬\|👇\|❌\|✅\|💥\|💎" "$dir"/slide-*.html
```

#### P1 · 触摸反馈（CRITICAL）

| 检查项 | 状态 |
|--------|------|
| Footer 按钮有 `:active` 态 | `transform: scale(0.96)` |
| Footer 按钮有 `cursor: pointer` | 可触达提示 |
| 触摸目标高度 | Footer span ≥ 44px（若不足用 padding 撑高） |

#### P2 · 视觉一致性

| 检查项 | 状态 |
|--------|------|
| **全系列背景统一**：暖白 `#FFF8F3` 或 深色 `#1a1a2e`，不能混用 | 封面+结尾用深色，内容页用暖白，形成首尾呼应 |
| 深色玻璃页（slide-02/07）用 Sans-serif 标题 | `Noto Sans SC` 而非 `Noto Serif SC` |
| bonus / think 视觉层级区分 | bonus 用极淡背景 + 金色左竖线；think 用 `#2D2D2D` 深色背景 |
| 装饰元素有物理质感 | deco 圆加 `border: 1px solid rgba(255,255,255,0.25)` 内折射边缘 |

#### P3 · 反 AI Slop

| 禁止项 | 说明 |
|--------|------|
| 无 Emoji | 所有 `👑💬👇❌✅💥💎` 必须替换为纯文字或 SVG |
| 无 Outer Glow | 文字 `text-shadow` 禁止 `0 4px 24px` 类发光效果，用 `0 2px 0 rgba(0,0,0,0.15)` 硬阴影 |
| 无多色杂乱 | 一个 slide 只用一个主 accent 色（珊瑚/金色选一） |

#### P4 · 排版细节

| 检查项 | 标准 |
|--------|------|
| footer `bottom` | ≥ 30px |
| footer `border-radius` | ≤ 10px |
| hook-end | `font-size: 13px` + `font-weight: 800` + `text-shadow` |
| hook-row `line-height` | 1.5 + `margin-bottom: 6px` |
| 深色页 `tag-note` alpha | ≥ 0.75 |

### 快速修复模板（Python 批量）

```python
files = [f"slide-0{i}-cover.html" for i in range(1,8)]
for fname in files:
    with open(fname, 'r', encoding='utf-8') as f:
        c = f.read()
    # P0 对比度
    c = c.replace('#E85A4F', '#C0392B')
    c = c.replace('#C9A227', '#A08020')
    # P1 footer:active
    if '.footer span:active' not in c:
        c = c.replace(
            '.footer span { ... }',
            '.footer span { cursor: pointer; transition: transform 0.15s ease; }\n.footer span:active { transform: scale(0.96); }'
        )
    # P2 bonus 淡化（仅 slide-06）
    if 'bonus' in fname:
        c = c.replace('rgba(201,162,39,0.18)', 'rgba(160,128,32,0.06)')
    # P3 emoji
    for old, new in [('👑','👑'),('💬','「'),('👇','↓'),('❌','X'),('✅','V'),('💥','!'),('💎','◆')]:
        c = c.replace(old, new)
    with open(fname, 'w', encoding='utf-8') as f:
        f.write(c)
```

**通用规则**（后续所有HTML模板必须遵守）：

### 2026-05-10 起号阶段禁止内容规范

**起号阶段（粉丝少、影响力弱）禁止出现的诱导互动内容**：

### 2026-05-09 手机端实际预览发现问题

**问题来源**：通过DingTalk截图在手机端实际预览「聊天总是聊死」第二集，发现以下问题：

5. **最小字号**：正文 ≥ 14px，标签 ≥ 13px

### body padding 导致底部裁切（2026-05-14 新增）

**问题描述**：所有幻灯片 HTML 文件的 `body { padding: 20px }` 导致 720px 的 slide 在 720px 视口中溢出，底部 footer 文字被裁切。

**根本原因**：`body { padding: 20px }` 让 body 实际渲染高度变成 720px + 40px（上下各 20px），但 viewport 是 720px，所以下方 20px 被截断。

**修复方法**：所有幻灯片 HTML 文件必须使用 `body { padding: 0 }`。

**强制规则**：

```css
/* 正确 */
body {
  padding: 0;
  display: flex;
  justify-content: center;
  align-items: center;
  min-height: 100vh;
}

/* 错误 — 会导致底部裁切 */
body {
  padding: 20px;  /* ← 禁止 */
}
```

**验证检查清单**：

```bash
# 检查所有幻灯片是否有 body padding
grep -rn "padding: 20px\|padding:20px" */幻灯片/slide-*.html

# 如果有输出，必须全部改为 padding: 0
```
