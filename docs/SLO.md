# 追爱AI SLO 定义

> 基于「生产就绪运维框架」的 Error Budget 机制

---

## 核心 SLI 定义

| SLI | 定义 | 测量方式 |
|-----|------|---------|
| **可用性** | API 请求成功比例 | HTTP 2xx / 总请求 |
| **延迟** | P99 响应时间 | 耗时排序后 99% 请求 |
| **错误率** | 5xx 错误比例 | HTTP 5xx / 总请求 |

---

## SLO 目标

| 接口 | 可用性 | P99 延迟 | 5xx 错误率 |
|------|--------|---------|-----------|
| `/api/version/check` | 99.9% | 200ms | < 0.1% |
| `/api/version/health` | 99.9% | 500ms | < 0.1% |
| `/api/auth/login` | 99.5% | 500ms | < 0.5% |
| `/api/chat/*` | 99.0% | 1000ms | < 1.0% |

---

## Error Budget

基于月可用性计算：

| SLI | 月 Error Budget |
|-----|-----------------|
| 99.9% | 43.8 分钟/月的不可用时间 |
| 99.5% | 3.65 小时/月的不可用时间 |
| 99.0% | 7.3 小时/月的不可用时间 |

---

## 发布决策规则

| Error Budget 状态 | 行动 |
|-------------------|------|
| **> 50% 剩余** | 可以快速发布，每天最多 3 次 |
| **20% - 50% 剩余** | 谨慎发布，每天最多 1 次 |
| **< 20% 剩余** | 暂停功能发布，专注可靠性 |
| **< 5% 剩余** | 立即停止发版，修补问题 |

---

## 告警阈值

| 指标 | 警告 | 紧急 |
|------|------|------|
| API 延迟 P99 | > 目标值 | > 2x 目标值 |
| API 错误率 | > 0.5% | > 1% |
| 数据库连接 | > 100ms | > 500ms |
| 内存使用 | > 70% | > 85% |
